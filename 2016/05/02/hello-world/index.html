发表于 2016-05-02   |   分类于 博客记

私人博客成功上线。

这里首先感谢一下 Jekyll迁移到Hexo搭建个人博客 与 Hexo 3.1.1 静态博客搭建指南 两篇博文，为本博客的搭建提供了参考。下面记录一下本次搭建遇到的两个小坑

博客搭建两小坑

两个问题都是同一个模块导致：dtrace-provider 模块无法正常安装

Python版本过新

博客是在mac系统上初始化，iTerm默认启动Python3虚拟环境，因为这个python版，dtrace-provider 模块死活无法安装，如果碰巧你也是这种情况，那么你在初始化hexo博客也一定会碰到下面错误： 

{ [Error: Cannot find module './build/Release/DTraceProviderBindings'] code: 'MODULE_NOT_FOUND' }  
{ [Error: Cannot find module './build/default/DTraceProviderBindings'] code: 'MODULE_NOT_FOUND' }
{ [Error: Cannot find module './build/Debug/DTraceProviderBindings'] code: 'MODULE_NOT_FOUND' }

自己配置好的python3虚拟环境，自然不想重新捣鼓过，从网上搜索找到了下面的配置方法： 

npm config set python python2.7  # 前提：python2.7 已经安装

配置好之后，已经初始化好的博客目录下的操作仍旧是会报上面的那个错误，解决方案： 

sudo npm install -g dtrace-provider  # 全局安装 dtrace-provider  
sudo npm install -g hexo-cli  # 重新全局安装 hexo-cli，无需卸载  
hexo init new_folder  # 重新初始化一个新的博客，删除掉旧的 ;)

dtrace-provider加载过程报找不到文件

这个错误比较诡异，报出这个错误，有可能是因为部署dtrace-provider模块过程中，shell界面卡了很久，最后被我ctrl+c强制退出了。 然后在hexo初始化（hexo init）过程中报出了找不到文件的错误。
详细的报错没有记录下来，大概的错误描述就是无法找到dtrace-provider安装目录下的compile.py文件，因此无法将其重命名为binding.gyp。
但是，该目录下已经有binding.gyp了，，，于是，我将binding.gyp复制一份命名为compile.py，重新加载，搞定。

关于本博客

学习了一段时间的编程，一直希望能够利用代码写出一个博客，一开始的设想是使用python+django的来完成一个博客系统的构建，当然现在大家看到了这个博客。。那么结局可想而知了。

为了博客而博客，通常这个博客搭建起来就会没有然后了。
这当然不是我想要的结局，因此打算将此博客作为记录学习编程过程中点点滴滴的输出渠道。

以此为勉。
